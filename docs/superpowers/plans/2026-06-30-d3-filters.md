# D3 — Search Filters Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task (the user has chosen **inline execution**). Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an Idealista-style filter bar to `/search` (price, beds, baths, property type, plus Florida filters: waterfront, pool, 55+, no-HOA) that applies live and updates the list, map, count, and URL — reusing all of the D2 search machinery, with the exposed filter set driven by an admin-configurable seam (not hardcoded).

**Architecture:** D2 already owns the synced list+map, the `searchListings` PostGIS query, the `SearchParams` URL contract, and the live `fetchFor` refetch. D3 **layers filters on top**: (1) extend the listings schema with three Florida booleans and add a `search_filters` config table (mirrors `qualification_questions` — admin edits it in D11); (2) extend the typed `SearchParams` wire format + `searchListings` predicates; (3) a code-side **filter registry** binds each configurable filter `key` to its typed param(s) + control type (the unsafe param→SQL mapping stays in code; the DB row drives only presentation); (4) a filter-bar UI of dropdown "pills" + a "More filters" panel with a live "See N homes" count; (5) a desktop Split/List layout toggle and a mobile filters sheet. Everything flows through the existing `fetchFor` → list + map + `?query` → SSR-friendly + shareable.

**Tech Stack:** Next.js 15 (Pages Router) + React 19 + TS strict; Drizzle + PostGIS (Neon); Zod boundaries; MapLibre (D2, untouched); vitest (pure-logic TDD); CSS Modules + design tokens.

## Global Constraints

- **Pages Router only** — no App Router, no server components, no `"use client"`. Client-only libs via `next/dynamic { ssr: false }` (already done for the map; filters render in the existing client tree).
- **TS strict** with `noUncheckedIndexedAccess` + `verbatimModuleSyntax`. Root `pnpm typecheck` covers `packages/*` only — **`apps/web` is type-checked by `pnpm --filter @herrera/web typecheck`** (run it for every web change).
- **Reuse, don't rebuild** the D2 machinery: `searchListings` (`packages/db/src/search.ts`), `SearchParams` + `parse/serializeSearchQuery` (`apps/web/src/lib/search-params.ts`), `runSearch`, `/api/search`, and `SearchView`'s `fetchFor`/`onViewportChange`. Filters compose with bbox + draw-zone, they do not replace them.
- **Filters are intentionally limited** (lead-gen — ADR-007/012): the exposed set is the eight below, no more. Leave reasons to contact Nilyan.
- **Admin-configurable seam, NOT hardcoded** (ADR-007): the filter SET (which filters, order, labels, options, bar-vs-panel, active) is rows in `search_filters`; the param/SQL binding is a code registry keyed by stable `key`. The admin editor UI is **D11** — build only the data model + seam now.
- **Live apply, no full reload:** filter changes call the existing client `fetch('/api/search?…')` path; `router.replace(..., { shallow: true })` keeps the URL shareable + SSR-friendly. Results count is reused from `SearchResults`.
- **Aesthetic stays ours** — design tokens only: `--color-forest #15302C`, `--color-paper #F3EFE7`, `--color-bronze #A9794A`, Spectral (display) + Hanken Grotesk (UI). Idealista is a **UX/pattern** reference only; never copy its visuals.
- **Mobile:** filters usable on the existing List/Map toggle layout (a "Filters" button → bottom sheet).
- **i18n seam:** every filter label carries `labelEs` (EN renders now; `/es` wiring is D13). Do not build the ES toggle.
- **Commits:** one per task on `feat/d3-filters`; every gate green before commit — `pnpm format:check && pnpm lint && pnpm --filter @herrera/web typecheck && pnpm test` (+ `pnpm --filter @herrera/web build` for tasks that touch web runtime/SSR). ## The exposed filter set (default `search_filters` seed)

| key | control | bar/panel | binds to `SearchParams` | SQL predicate |
|---|---|---|---|---|
| `price` | `range` | bar | `minPrice`, `maxPrice` | `l.price >= min`, `l.price <= max` |
| `beds` | `min_select` | bar | `minBeds` | `l.bedrooms >= n` |
| `baths` | `min_select` | bar | `minBaths` (NEW) | `l.bathrooms >= n` |
| `propertyType` | `enum_select` (multi) | bar | `types` (NEW, replaces single `type`) | `l.property_type IN (…)` |
| `waterfront` | `boolean` | panel | `waterfront` (NEW) | `l.waterfront = true` |
| `pool` | `boolean` | panel | `pool` (NEW) | `l.pool = true` |
| `age55` | `boolean` | panel | `age55` (NEW) | `l.age_restricted = true` |
| `noHoa` | `boolean` | panel | `noHoa` (NEW) | `l.hoa_fee_monthly IS NULL OR = 0` |

`waterfront`/`pool`/`age_restricted` are **new listings columns**; `noHoa` reuses the existing `hoa_fee_monthly`.

- [ ] **Step 5: Create `packages/db/src/schema/search-filters.ts`**

```ts
import { sql } from "drizzle-orm";
import {
  boolean,
  index,
  integer,
  jsonb,
  pgTable,
  text,
  timestamp,
  uniqueIndex,
  uuid,
} from "drizzle-orm/pg-core";
import { createInsertSchema, createSelectSchema } from "drizzle-zod";
import { z } from "zod";
import { searchFilterControl } from "./enums";
import { questionOptionSchema, type QuestionOption } from "./json";

// ADR-007/012 (D3) — the admin-configurable SET of filters exposed on /search. `key` binds to a
// code-side registry (param parsing + PostGIS predicate, injection-safe). This row drives only
// PRESENTATION: which filters show, order, labels, options, bar-vs-"More filters", active. Nilyan
// edits these in /admin in D11; intentionally limited (lead-gen — ADR-007/012).
export const searchFilters = pgTable(
  "search_filters",
  {
    id: uuid("id").primaryKey().defaultRandom(),
    key: text("key").notNull(), // stable code key (price, beds, baths, propertyType, waterfront, pool, age55, noHoa)
    control: searchFilterControl("control").notNull(),
    sortOrder: integer("sort_order").notNull().default(0),
    label: text("label").notNull(),
    labelEs: text("label_es"),
    options: jsonb("options")
      .$type<QuestionOption[]>()
      .notNull()
      .default(sql`'[]'::jsonb`),
    advanced: boolean("advanced").notNull().default(false), // true → "More filters" panel
    isActive: boolean("is_active").notNull().default(true),
    createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
    updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
  },
  (t) => [
    uniqueIndex("search_filters_key_uq").on(t.key),
    index("search_filters_order_ix").on(t.sortOrder),
  ],
);

export const searchFilterInsertSchema = createInsertSchema(searchFilters, {
  options: z.array(questionOptionSchema).optional(),
});
export const searchFilterSelectSchema = createSelectSchema(searchFilters);
export type SearchFilter = typeof searchFilters.$inferSelect;
export type NewSearchFilter = typeof searchFilters.$inferInsert;
```

```ts
import type { SearchFilterControl } from "@herrera/db/schema/enums"; // if not re-exported, inline the union
import type { PropertyType, SearchParams } from "./search-params";

// The union is small + stable; inline to avoid a deep import if the above isn't exported:
export type FilterControl = "range" | "min_select" | "enum_select" | "boolean";

export type FilterConfig = {
  key: string;
  control: FilterControl;
  label: string;
  labelEs: string | null;
  options: { value: string; label: string; labelEs?: string | null }[];
  advanced: boolean;
};

export type FilterBinding = {
  control: FilterControl;
  isSet: (p: SearchParams) => boolean;
  clear: (p: SearchParams) => SearchParams;
};

const omit = (p: SearchParams, ...keys: (keyof SearchParams)[]): SearchParams => {
  const next = { ...p };
  for (const k of keys) delete next[k];
  return next;
};

const boolBinding = (key: keyof SearchParams): FilterBinding => ({
  control: "boolean",
  isSet: (p) => p[key] === true,
  clear: (p) => omit(p, key),
});

// key → typed-param binding. Unknown keys (not here) are ignored by the UI = injection-safe seam.
export const FILTER_BINDINGS: Record<string, FilterBinding> = {
  price: {
    control: "range",
    isSet: (p) => p.minPrice != null || p.maxPrice != null,
    clear: (p) => omit(p, "minPrice", "maxPrice"),
  },
  beds: {
    control: "min_select",
    isSet: (p) => p.minBeds != null,
    clear: (p) => omit(p, "minBeds"),
  },
  baths: {
    control: "min_select",
    isSet: (p) => p.minBaths != null,
    clear: (p) => omit(p, "minBaths"),
  },
  propertyType: {
    control: "enum_select",
    isSet: (p) => (p.types?.length ?? 0) > 0,
    clear: (p) => omit(p, "types"),
  },
  waterfront: boolBinding("waterfront"),
  pool: boolBinding("pool"),
  age55: boolBinding("age55"),
  noHoa: boolBinding("noHoa"),
};

export function activeFilterKeys(p: SearchParams, configs: FilterConfig[]): string[] {
  return configs.filter((c) => FILTER_BINDINGS[c.key]?.isSet(p)).map((c) => c.key);
}
export function activeFilterCount(p: SearchParams, configs: FilterConfig[]): number {
  return activeFilterKeys(p, configs).length;
}

// Value read/write helpers used by FilterControl (kept here so the param shape lives in one place).
export const readMin = (p: SearchParams, key: "minBeds" | "minBaths") => p[key];
export const writeMin = (key: "minBeds" | "minBaths", v: number | undefined): SearchParams =>
  v == null ? {} : ({ [key]: v } as SearchParams);
export const readTypes = (p: SearchParams): PropertyType[] => p.types ?? [];
```

```tsx
import { useEffect, useId, useRef, useState, type ReactNode } from "react";
import styles from "./Popover.module.css";

export function Popover({
  label,
  badge,
  active,
  children,
}: {
  label: string;
  badge?: string;
  active?: boolean;
  children: (close: () => void) => ReactNode;
}) {
  const [open, setOpen] = useState(false);
  const ref = useRef<HTMLDivElement>(null);
  const id = useId();
  useEffect(() => {
    if (!open) return;
    const onDown = (e: MouseEvent) => {
      if (ref.current && !ref.current.contains(e.target as Node)) setOpen(false);
    };
    const onKey = (e: KeyboardEvent) => e.key === "Escape" && setOpen(false);
    document.addEventListener("mousedown", onDown);
    document.addEventListener("keydown", onKey);
    return () => {
      document.removeEventListener("mousedown", onDown);
      document.removeEventListener("keydown", onKey);
    };
  }, [open]);
  return (
    <div className={styles.wrap} ref={ref}>
      <button
        type="button"
        className={`${styles.trigger} ${active ? styles.active : ""}`}
        aria-expanded={open}
        aria-controls={id}
        onClick={() => setOpen((o) => !o)}
      >
        {label}
        {badge ? <span className={styles.badge}>{badge}</span> : null}
        <span aria-hidden className={styles.caret}>▾</span>
      </button>
      {open ? (
        <div id={id} role="dialog" className={styles.panel}>
          {children(() => setOpen(false))}
        </div>
      ) : null}
    </div>
  );
}
```

```tsx
export function FilterControl({

  onChange,

  draft: SearchParams;

}) {
  if (config.control === "range") {
    return (
      <div>
        <label>
          Min{" "}
          <input
            type="number"
            inputMode="numeric"

          />

        <label>
          Max{" "}
          <input

            inputMode="numeric"
            value={draft.maxPrice ?? ""}
            onChange={(e) => onChange({ ...draft, maxPrice: e.target.value ? Number(e.target.value) : undefined })}
          />
        </label>
      </div>
    );
  }
  if (config.control === "min_select") {
    const key = config.key === "baths" ? "minBaths" : "minBeds";
    const current = draft[key];
    return (
      <div role="group" aria-label={config.label}>
        {[undefined, ...config.options].map((opt) => {
          const v = opt ? Number(opt.value) : undefined;
          const selected = current === v;
          return (
            <button
              type="button"
              key={opt?.value ?? "any"}
              aria-pressed={selected}
              onClick={() => onChange({ ...draft, [key]: v })}
            >
              {opt ? opt.label : "Any"}
            </button>
          );
        })}
      </div>
    );
  }
  if (config.control === "enum_select") {
    const selected = new Set<PropertyType>(draft.types ?? []);
    return (
      <div role="group" aria-label={config.label}>
        {config.options.map((opt) => {
          const val = opt.value as PropertyType;
          if (!(PROPERTY_TYPES as readonly string[]).includes(val)) return null;
          const on = selected.has(val);
          return (
            <label key={opt.value}>
              <input
                type="checkbox"
                checked={on}
                onChange={() => {
                  const next = new Set(selected);
                  on ? next.delete(val) : next.add(val);
                  onChange({ ...draft, types: next.size ? [...next] : undefined });
                }}
              />
              {opt.label}
            </label>
          );
        })}
      </div>
    );
  }
  // boolean
  const bkey = config.key as "waterfront" | "pool" | "age55" | "noHoa";
  return (
    <label>
      <input
        type="checkbox"
        checked={draft[bkey] === true}
        onChange={(e) => onChange({ ...draft, [bkey]: e.target.checked ? true : undefined })}
      />
      {config.label}
```

- [ ] **Step 4: `MoreFiltersPanel.tsx`** — advanced filters + live "See N homes" preview.

```tsx
import { useEffect, useState } from "react";
import { serializeSearchQuery, type SearchParams } from "@/lib/search-params";
import { FilterControl } from "./FilterControl";
import type { FilterConfig } from "@/lib/filters";
import styles from "./MoreFiltersPanel.module.css";

export function MoreFiltersPanel({
  filters,
  params,
  open,
  total,
  onClose,
  onApply,
}: {
  filters: FilterConfig[];
  params: SearchParams;
  open: boolean;
  total: number;
  onClose: () => void;
  onApply: (p: SearchParams) => void;
}) {
  const [draft, setDraft] = useState<SearchParams>(params);
  const [preview, setPreview] = useState<number>(total);
  useEffect(() => {
    if (open) {
      setDraft(params);
      setPreview(total);
    }
  }, [open, params, total]);

  // Live count preview as the user toggles (debounced count-only fetch reusing /api/search).
  useEffect(() => {
    if (!open) return;
    const ctrl = new AbortController();
    const t = setTimeout(async () => {
      try {
        const qs = new URLSearchParams(serializeSearchQuery(draft)).toString();
        const res = await fetch(`/api/search?${qs}`, { signal: ctrl.signal });
        const data = await res.json();
        setPreview(data.total ?? 0);
      } catch {
        /* aborted */
      }
    }, 350);
    return () => {
      clearTimeout(t);
      ctrl.abort();
    };
  }, [draft, open]);

  if (!open) return null;
  return (
    <div className={styles.overlay} role="dialog" aria-label="More filters" aria-modal="true">
      <div className={styles.sheet}>
        <header className={styles.head}>
          <h2>More filters</h2>
          <button type="button" onClick={onClose} aria-label="Close">×</button>
        </header>
        <div className={styles.body}>
          {filters.map((f) => (
            <section key={f.key} className={styles.row}>
              <FilterControl config={f} draft={draft} onChange={setDraft} />
            </section>
          ))}
        </div>
        <footer className={styles.foot}>
          <button
            type="button"
            className={styles.reset}
            onClick={() => setDraft((d) => clearAdvanced(d, filters))}
          >
            Reset
          </button>
          <button
            type="button"
            className={styles.apply}
            onClick={() => {
              onApply(draft);
              onClose();
            }}
          >
            See {preview} {preview === 1 ? "home" : "homes"}
          </button>
        </footer>
      </div>
    </div>
  );
}

function clearAdvanced(p: SearchParams, filters: FilterConfig[]): SearchParams {
  const next = { ...p };
  for (const f of filters) delete next[f.key as keyof SearchParams];
  return next;
}
```

CSS: `.overlay` desktop = right-anchored panel (`position: absolute; inset: 0 0 0 auto; width: min(420px, 92vw)`) with forest-tinted scrim behind; mobile (`@media max-width: 900px`) = bottom sheet (`inset: auto 0 0 0; max-height: 86dvh; border-radius: 16px 16px 0 0`). `.apply` = bronze button; respects reduced-motion (no slide animation when `prefers-reduced-motion`).
